How it works

Three steps, all from the page image — the model never reads the text.

1

Upload a page

Drop a scanned German document page (JPG/PNG). It's resized to 224×224 and normalized.

2

ResNet50 reads the layout

A fine-tuned ResNet50 CNN analyzes the visual structure — margins, blocks, headers, whitespace.

3

Get a typed result

You receive the predicted category with a confidence score and a breakdown across all six types.

What makes it different

👁️

Layout, not text

Classifies by visual structure rather than OCR'd words — so it works on noisy scans and is language-robust.

🧠

Transfer learning

ResNet50 pretrained on ImageNet, then two-stage fine-tuned on German document pages.

⚡

Lightweight serving

Runs on a TensorFlow Lite runtime — small footprint, CPU-only, deployable on a free tier.

🔌

Simple API

A single POST /predict endpoint returns the class, confidence, and full probability vector.

Under the hood

ModelResNet50 (transfer learning, two-stage fine-tune)
DatasetGerman subset of DocLayNet
Input224×224 RGB page image
RuntimeTensorFlow Lite · FastAPI
Output6-class softmax + confidence

Good to know: the model is trained specifically on German document pages across these six types and always returns one of them. Feeding it PDFs, English documents, photos, or other document kinds will produce a confident but likely incorrect label — that's expected, not a bug.
